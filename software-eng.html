Programming Languages

Term Definition
Software Engineering the branch of computer science that seeks principles to guide the development of large, complex software systems.
Integrated Development Environment(IDE) Combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.
Association for Computing Machinery(ACM)
  • Purpose: educational organization dedicated to advancing the arts, sciences, and applications of information technology.
  • Member benefits: provides lifelong learning programs, original research and content, and opportunities to meet peers.
  • Student membership: provides the benefits of being a member of ACM for a large discount to students within one year after graduation
Institute of Electrical and Electronics Engineers(IEEE)
  • Purpose: The mission of the IEEE is advancing technology for the benefit of humanity.
  • Membership Benefits: Provides access to mass amounts of documents and pograms to work and collaborate with other members on
  • Student membership: Provides the same advantages as a regular member at a highly discounted cost
Software Life Cycle: Development phase
  • Requirements analysis: specify what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
  • Design: involves creating a plan for the construction of the proposed system.
  • Implementation: the actual writing of programs, creation of data files, and development of databases.
  • Testing: the process of debugging programs and confirming that the final software product was compatible with the software requirements specification.
Modular Implementation Seperates the code into modules, and allows parts of the code to be implemented in an order
Unified Modeling Language(UML) More modern collection of tools developed with object-oriented paradigm in mind
Use case Interactions between the system and its users
Use case diagram Depicts the interactions of use cases and actors in a system
Data relationships:
  • one-to-one: Entities of type X only interact with one type Y
  • one-to-many: Entities of type X interact with multiple type Y
  • many-to-many: Entities of both types interact with many of the others
Design Pattern Predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) Charged with overseeing and enforcing the quality control systems adopted by the organization.
Software Testing:
  • Glass box testing: The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
  • Black box testing: Tests that do not rely on knowledge of the software’s interior composition.
  • Beta testing: A preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
  • Alpha testing: A form of beta testing that is performed at the developer's site
Documentation
  • User documentation: Explain the features of the software and describe how to use them
  • System documentation: Describe the software’s internal composition so that the software can be maintained later in its life cycle.
  • Technical documentation: Describe how a software system should be installed and serviced (such as adjusting operating parameters, installing updates, and reporting problems back to the software’s developer).
Intellectual Property Law:
  • Software License: Legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
  • Software disclaimer: Developers denial of liability for consequences one may have when using the program
  • Best defenses against liability claims: Apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors.
